# Qwen-VL 系列与动态 FPS 视频

> Qwen-VL 系列——Qwen-VL（2023）、Qwen2-VL（2024）、Qwen2.5-VL（2025）、Qwen3-VL（2025）——是 2026 年最具影响力的开源视觉语言模型谱系。每一代都做了一个决定性的架构押注，开源生态在十二个月内就纷纷效仿：通过 M-RoPE 实现原生动态分辨率、带绝对时间对齐的动态 FPS 采样、ViT 中的窗口注意力、以及结构化代理输出格式。到 Qwen3-VL，方案已趋于稳定：带有原生宽高比输入的 2D-RoPE-ViT 编码器、MLP 投影器接入大型 Qwen3 语言基座、以及将 OCR、定位和代理行为作为一等目标的训练阶段。本课按时间线解读这个系列，让你理解每个旋钮为什么在那里。

**类型：** Learn
**语言：** Python（标准库，M-RoPE 编码器 + 动态 FPS 采样器）
**前置课程：** Phase 12 · 06（patch-n'-pack）
**时间：** ~120 分钟

## 学习目标

- 计算 M-RoPE 的三轴旋转（时间、高度、宽度）并解释为什么三者都需要。
- 为视频选择动态 FPS 采样策略，并推理每秒 token 数 vs 事件检测准确率。
- 按顺序说出 Qwen-VL 的四代升级及其各自带来的能力。
- 接入 Qwen2.5-VL 风格的 JSON 代理输出格式，并从 VLM 响应中解析结构化工具调用。

## 问题

Qwen-VL 于 2023 年 8 月发布，是对 LLaVA-1.5 和 BLIP-2 的直接回应。Qwen 团队瞄准的差距在三个方面：分辨率、视频和结构化输出。

分辨率：LLaVA-1.5 在 336×336 下运行。对照片够用，对中文发票或密集电子表格截图无用。Qwen-VL 的第一个创新是 448×448 和定位边界框输出，让模型可以指向事物。

视频：Video-LLaMA 堆叠逐帧编码器并将它们输入 LLM。对短视频有效，对时间轴是信号的多分钟视频无效。Qwen 团队想要一个理解时间的单一编码器。

结构化输出：LLaVA 输出自由形式文本。代理需要 JSON。Qwen-VL 在显式 JSON 输出格式上训练，包括将边界框坐标作为文本。

每一代 Qwen-VL 都延伸了这三条轴之一。

## 概念

### Qwen-VL（2023 年 8 月）

第一代：OpenCLIP ViT-bigG/14 作为编码器（25 亿参数），LLama 兼容的 Q-Former（1 步 256 个 query），Qwen-7B 基座。贡献：

- 448×448 分辨率（当时开源 VLM 的 SOTA）。
- 定位（Grounding）：在带有显式坐标 token 输出的图文对上训练。"猫在 <box>(112, 204), (280, 344)</box>"。
- 从一开始就中英双语训练。

当时的基准：英语与 GPT-4V 竞争力相当，中文占主导。定位监督是真正的亮点。

### Qwen2-VL（2024 年 9 月）——M-RoPE 和原生分辨率

Qwen2-VL 用原生动态分辨率 ViT 编码器替换了固定分辨率 + Q-Former 堆栈。关键变化：

- **原生动态分辨率**。ViT 接受任何能被 28 整除的 H×W（patch 14 配合 2× 空间合并）。1120×672 的图像（40×24 合并 patch）产生 960 个视觉 token。没有调整大小，没有拼贴，没有缩略图。
- **M-RoPE（多模态 RoPE）**。每个 token 携带 3D 位置 (t, h, w) 而非 1D。图像 t=0，视频 t=帧索引。RoPE 按每轴频率旋转 query/key 向量。无位置嵌入表。
- **MLP 投影器**。去掉 Q-Former；对合并的 patch token 使用 2 层 MLP。
- **动态 FPS 视频**。默认以 1-2 FPS 采样视频，但模型接受任意帧数。

结果：Qwen2-VL-7B 在多个多模态基准上匹配 GPT-4o，在 DocVQA 上超越（94.5 vs 88.4）。架构变化是决定性举措。

### Qwen2.5-VL（2025 年 2 月）——动态 FPS + 绝对时间

Qwen2.5-VL 的重大转变在视频方面。动态 FPS 不仅仅是"需要时采样更多帧"。论文形式化了：

- **绝对时间 token**。不用位置索引（帧 0、1、2...），而用实际时间戳。"在 0:04，猫跳了。"模型看到 `<time>0.04</time>` token 与帧 token 交错。
- **动态 FPS**。慢镜头以 1 FPS 采样，动作场景以 4+ FPS 采样。用户或训练器选择；M-RoPE 自适应。
- **ViT 窗口注意力**。空间注意力是窗口式的（块内局部）以提高吞吐量；每隔几层加全局注意力。
- **显式 JSON 输出格式**。在工具调用数据上训练：`{"tool": "click", "coords": [380, 220]}`。开箱即用的代理就绪。
- **MRoPE-v2 缩放**。位置随最大输入大小缩放，使 10 分钟视频不会用完频率范围。

基准：Qwen2.5-VL-72B 在大多数视频基准上击败 GPT-4o，在文档上匹配 Gemini 2.0，并为 GUI 定位设立了开源模型 SOTA（ScreenSpot：84% 准确率 vs GPT-4o 的 38%）。

### Qwen3-VL（2025 年 11 月）

Qwen3-VL 是增量升级，巩固而非重新发明：更大的 LLM 骨干网络（Qwen3-72B）、扩展的训练数据、改进的 OCR、通过 Qwen3"思考模式"增强的推理。ViT 和 M-RoPE 保持不变。论文聚焦于数据和训练改进而非架构。

谱系要点：到 2025 年，Qwen-VL 架构已趋于稳定。后续代次扩展计算和数据，而非原语。

### M-RoPE 的数学

经典 RoPE 使用配对坐标按位置 `m` 旋转维度 `d` 的 query `q`：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE 将隐藏维度分成三个频段。假设 `d = 96`。分配 32 维给时间、32 给高度、32 给宽度。每个频段按自己的轴位置旋转。位置 (t=5, h=10, w=20) 的 patch 获得旋转 `R_t(5)`、`R_h(10)`、`R_w(20)` 分别应用于三个频段。

文本 token 使用 `t = text_index, h = 0, w = 0`（或归一化选择），保持兼容性。视频帧使用 `t = frame_time, h = row, w = col`。单图像使用 `t = 0`。

好处：一种位置编码处理文本、图像和视频，无需分支代码或不同的位置表。

### 动态 FPS 采样逻辑

给定时长 `T` 秒的视频和目标 token 预算 `B`：

1. 计算你能承受的最大 FPS：`fps_max = B / (T * tokens_per_frame)`。
2. 从 `{1, 2, 4, 8}` 中选择满足 `fps <= fps_max` 的目标 FPS。
3. 如果运动幅度大（光流启发式或用户明确请求），选择更高 FPS。如果运动幅度小，选择更低。
4. 以选定 FPS 均匀采样；在帧之间插入 `<time>t</time>` token。

Qwen2.5-VL 隐式训练此逻辑；推理时用户通过 `fps` 参数控制。60 秒的动作序列以 4 FPS 每帧 81 个 token = 19440 个 token，在 32k 上下文中可控。

### 结构化代理输出

Qwen2.5-VL 的代理训练显式针对结构化工具调用：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析是确定性的：对模型输出做 JSON.parse。对比自由形式的"click at (1024, 512)"需要正则表达式和歧义处理。这就是 Qwen2.5-VL 的 ScreenSpot 分数从 Qwen2-VL 的 55% 跳到 84% 的原因。

- 对混合文本、图像 patch 和视频帧的打包序列计算 M-RoPE 位置。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| M-RoPE | "多模态 RoPE" | 隐藏维度中具有时间、高度和宽度频段的 3D 旋转位置嵌入 |
| Dynamic FPS（动态 FPS） | "智能采样" | 根据运动、时长和 token 预算为每个视频选择的帧采样率 |
| Absolute time token（绝对时间 token） | "时间戳 token" | 交错在序列中的 `<time>t</time>`，让模型看到实际秒数而非帧索引 |
| Window attention（窗口注意力） | "局部注意力" | 限制在小窗口内的空间自注意力以提高速度；周期性添加全局注意力 |
| Structured agent output（结构化代理输出） | "JSON 模式" | 教 VLM 发出带坐标和工具名的可解析 JSON 的训练数据监督 |
| min_pixels / max_pixels | "分辨率界限" | Qwen2.5-VL 的每请求控制，限制总像素数从而限制 token 数 |
| Grounding（定位） | "指向它" | 将边界框坐标作为文本 token 输出；自 Qwen-VL v1 开始使用 |
