# InternVL3：原生多模态预训练

> InternVL3 之前的每个开源 VLM 都遵循相同的三步方案：取一个在数万亿文本 token 上训练的文本 LLM，接上视觉编码器，然后微调接缝处。这可行但存在对齐债务——文本 LLM 已将其全部预训练预算花在纯文本上，并不原生理解视觉 token。当你事后添加视觉时，LLM 必须在不遗忘文本的情况下重新学习如何将视觉输入与其文本推理关联。InternVL3（Zhu 等人，2025 年 4 月）拒绝了事后方法：一次预训练运行，从第一步就交错文本和多模态。结果在 78B 参数开源下匹配 Gemini 2.5 Pro 的 MMMU-Pro。本课解读原生预训练的理由以及做出这一改变时发生的变化。

**类型：** Learn
**语言：** Python（标准库，训练语料混合器）
**前置课程：** Phase 12 · 05、Phase 12 · 07（方案）
**时间：** ~120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会累积对齐债务，引用三个可测量的症状（灾难性遗忘、回答漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料混合比例以及为什么文本：交错：描述的比率很重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出视觉分辨率路由器（ViR）和解耦视觉-语言（DvD）部署优化。

## 问题

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——都取一个已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据上训练（LLaVA-Instruct、ShareGPT4V）。
3. 可选的任务特定微调。

三个对齐债务的症状出现：

- **灾难性遗忘**。事后 VLM 忘记纯文本技能。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本代理退化。
- **回答漂移**。同一视觉问题的微小措辞变化得到不同答案。视觉编码器与 LLM 的连接比 LLM 自己的 token 更弱。
- **视觉-文本不一致**。VLM 可以正确描述图像，然后回答一个与其自身描述矛盾的问题。视觉 token 不像文本那样参与 LLM 的内部一致性检查。

这些症状有充分记录。MM1.5 第 4 节量化了它们。LLaVA-OneVision 的消融暗示了它们。原生预训练是答案。

## 概念

### 原生多模态预训练

InternVL3 从头开始在第一步就是原生多模态的语料上训练。混合比例为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 配对图文描述数据
- 5% 视频文本数据

视觉 token、文本 token 和跨模态交互从第一个梯度步骤就参与相同的损失。没有对齐预训练，没有投影器冻结阶段，没有需要恢复的灾难性遗忘。

基础模型训练是单阶段的。指令微调随之进行，但基础模型已经将视觉 token 理解为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用固定轴分配的 M-RoPE。InternVL3 引入 V2PE：位置编码按模态类型（文本、图像、视频）变化，具有可学习的缩放。实际中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享相同的 RoPE 频率基础，但每频段的隐藏维度分配是可学习参数而非固定分割。可在预训练期间自由权衡时间 vs 空间频率分辨率。

V2PE 的消融声明：在相同计算下比 M-RoPE 在视频基准上高 1-2 分。不是革命，但更干净。

### 视觉分辨率路由器（ViR）

部署优化。不是所有图像都需要全分辨率编码。一张低细节单物体照片在 1280px 原生分辨率下编码会浪费 token。ViR 是一个小分类器，在编码前预测回答问题所需的最小分辨率。

路由有三个层级：低分辨率（256 token）、中等（576）、高（2048+）。在生产流量中 60% 的查询，低或中等就足够了。净效果：同等质量下 2-3 倍吞吐量。

### 解耦视觉-语言部署（DvD）

当你服务大型 VLM 时，视觉编码器每张图像运行一次，但 LLM 对每个输出 token 自回归运行。两个组件有不同的瓶颈（视觉 = 卷积 + 注意力的 GPU 显存带宽；LLM = KV 缓存）。DvD 将它们分配到不同的 GPU 上，中间用流式传输。

对于 8B + 400M 编码器模型，DvD 相比共置部署大致使每节点吞吐量翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准声明：78B 参数下匹配 Gemini 2.5 Pro 的 MMMU-Pro。38B 下匹配 GPT-4o。8B 下领跑开源 8B 排行榜。全部基于单阶段预训练 + 指令微调方案。

对齐债务假说是可测量的：InternVL3-8B 在每单位视觉基准提升中损失更少的文本基准分数（MMLU、GSM8K）相比 Qwen2.5-VL-7B。模型更通用，因为训练是一体的，而非两段。

### InternVL3.5 和 InternVL-U

InternVL3.5（2025 年 8 月）扩展了方案。相同的原生预训练方法，更多数据，更多参数。MMMU 改进是增量的。

InternVL-U（2026）添加了统一生成——通过在相同骨干网络上的 MMDiT 头实现图像输出。"U"代表"理解 + 生成"，追随 Transfusion 风格的统一模型（课程 12.13）。相同的原生预训练骨干网络支持理解和生成头。

### 原生预训练的权衡

原生预训练不是免费的：

- **计算**。从头训练新 VLM 与训练文本 LLM 成本相同——数百万 GPU 小时。事后适配复用现有 LLM 权重，节省大部分成本。
- **数据**。大规模交错图文语料稀少。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。仅文本就有 15T token。多模态预训练数据稀缺是硬约束。
- **基座 LLM 复用**。原生预训练放弃了日后替换新 LLM 的选项。事后方法让你通过仅重新训练适配器将 Llama-3.1 换成 Llama-4。

InternVL3 的赌注：对齐债务比复用损失更严重。基准支持了这一声明。生产成本阻止了未来实验室廉价复现。事后 VLM 将继续存在，因为对大多数项目它们仍然更便宜。

## 使用它

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器。它：

- 接收目标语料混合比例（%文本、%交错、%描述、%视频）并计算每模态的预期步数。
- 在一批查询上模拟 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节）并报告平均 token 数。
- 给定编码器 vs LLM 的 FLOPs 报告 DvD 吞吐量估算。
- 打印事后 vs 原生预训练在参数、计算、数据和预期对齐债务症状上的对比。

## 练习

1. 估算 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后）之间的计算差异。GPU 小时比率大约是多少？差距的原因是什么？

2. InternVL3 报告 40% 文本 / 35% 交错 / 20% 描述 / 5% 视频。如果你的目标任务是视频密集型，提出一个新比例并论证为什么基础模型仍然需要大量文本和描述数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练显示最大退化的确切基准。退化代价是多少？

4. ViR 将 60% 流量路由到低分辨率编码。哪些查询会被错误路由（需要高分辨率时发送到低分辨率）？提出三种路由器失败模式。

5. DvD 将视觉和 LLM 分配到不同 GPU。在什么流量模式下 DvD 会损害吞吐量而非帮助？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Native multimodal pretraining（原生多模态预训练） | "从头一起训练" | 文本 + 图像 + 视频 token 从第 1 步就参与损失，而非事后嫁接 |
| Alignment debt（对齐债务） | "事后惩罚" | 将视觉嫁接到冻结 LLM 上导致的文本技能和回答一致性的可测量退化 |
| V2PE | "可变视觉位置编码" | 每模态可学习的位置编码分配；InternVL3 的 M-RoPE 后继 |
| ViR | "分辨率路由器" | 在编码前为每个查询选择最小所需分辨率的小型分类器，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一个 GPU 上，LLM 在另一个上，流式交接；使大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 后续，在原生预训练骨干网络上添加图像生成头 |
| Interleaved corpus（交错语料） | "OBELICS / MMC4" | 按自然阅读顺序排列的文本和图像文档；原生预训练的原材料 |
