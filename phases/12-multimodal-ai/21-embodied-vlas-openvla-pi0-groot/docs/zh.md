# 具身 VLA：RT-2、OpenVLA、π0、GR00T

> 模型第一次从网站读取食谱并在厨房机器人中执行是 RT-2（Google DeepMind，2023 年 7 月）。RT-2 将动作离散化为文本 token，在网络数据加机器人动作数据上联合微调 VLM，并证明了网络规模的视觉-语言知识可迁移到机器人控制。OpenVLA（2024 年 6 月）发布了开源 7B 参考模型。Physical Intelligence 的 π0 系列（2024-2025）添加了流匹配动作专家。NVIDIA 的 GR00T N1（2025 年 3 月）为大规模人形机器人提供了双系统（System 1 / System 2）控制。VLA 原语——视觉-语言-动作，一个能看、读和行动的单一模型——是本阶段理解模型与 Phase 15 自主系统之间的桥梁。

**类型：** Learn

## 学习目标

- 描述动作 token 化：离散分箱编码（RT-2）、FAST 高效动作 token、连续流匹配动作（π0）。
- 解释为什么在网络 + 机器人数据上联合微调能保留通用知识迁移到新任务。
- 对比 OpenVLA（开源 7B Llama+VLM）、π0（流匹配）和 GR00T N1（双系统）在相同机器人任务上的表现。
- 说出 Open X-Embodiment 数据集及其作为 RT-X 训练语料的角色。

## 问题

一个能从自然语言指令做家务的机器人自 1970 年代以来一直是研究目标。2020 年代的答案：视觉-语言-动作（VLA）模型。与 VQA 使用的 VLM 架构相同，但输出是动作（关节力矩、末端执行器位姿、离散命令）而非文本。

VLA 特有的挑战：

1. **动作空间是连续的**（关节角度、力）且高维的（7-DOF 手臂 + 3-DOF 夹爪 = 10 维，30 Hz）。
2. **机器人特定训练数据稀缺**。Open X-Embodiment 有约 100 万条轨迹；网络文本-图像有 50 亿+。
3. **控制频率很重要**。30 Hz 控制循环意味着每个动作 33ms 预算。
4. **安全**。错误动作会损坏硬件、人类或财产。

## 概念

### 动作 token 化（RT-2）

RT-2 的技巧：将每个关节目标表示为量化的文本 token。将归一化的 [-1, 1] 范围离散化为 256 个分箱，将每个分箱映射到词汇 ID。一个 10-DOF 动作在每个控制步骤变成 10 个 token。

在混合数据上联合微调 PaLM-X VLM：

- 网络图文对（描述、VQA）。
- 机器人演示，动作作为 token。

模型看到"拿起红色方块"（语言）→ 图像（视觉）→ 10 个 token 的动作序列（离散化关节目标）。网络预训练保留通用知识迁移：RT-2 可以遵循"移向快速移动的物体"，即使"快速移动"不在训练数据中。

RT-2 论文中推理速度 3-5 Hz，受限于 VLM 自回归解码。

### OpenVLA——开源 7B 参考模型

OpenVLA（Kim 等人，2024 年 6 月）是开源权重的 RT-2 等价物。7B Llama 骨干网络，DINOv2 + SigLIP 双视觉编码器，256 分箱的动作 token 化。

在 Open X-Embodiment（22 种机器人的 97 万条轨迹）上训练。发布时附带 LoRA 微调支持以适配新机器人。

推理：A100 上量化后 4-5 Hz。足够慢速操作，不够高频控制。

### FAST tokenizer——更快的动作解码

Pertsch 等人（2024）表明离散分箱 token 化效率低下——大多数动作聚集在分箱空间的小区域内。FAST（频域动作序列 tokenizer）通过 DCT 压缩动作序列并量化系数。

30 步动作轨迹变成约 10 个 FAST token 而非 300 个离散分箱 token。推理加速 3-5 倍且无质量损失。

### π0 和流匹配动作

Physical Intelligence 的 π0（Black 等人，2024 年 10 月）用流匹配动作专家替换了离散动作 token：

- 一个小型动作 Transformer 读取 VLM 的隐藏状态，通过整流流输出连续的 50 步动作序列。
- 动作头以流匹配损失训练；VLM 预训练保持不变。
- 推理：约 5 步去噪发出完整动作序列，有效 50 Hz 控制。

π0 的声明：在广泛的操作任务套件上击败 OpenVLA 和 Octo。连续动作公式保留了离散化破坏的平滑性。

π0.5 和 π0-FAST 是增量升级。π0-FAST 将 FAST token 化与流匹配结合。

### GR00T N1——人形机器人的双系统

NVIDIA 的 GR00T N1（2025 年 3 月）为（>30 DOF、全身）人形机器人构建：

- **System 2**：大型 VLM 读取场景 + 指令，以约 1 Hz 产生高层子目标。
- **System 1**：小型动作头 Transformer 以子目标为条件产生低层 50-100 Hz 关节命令。

该分离映射到卡尼曼的快思考和慢思考：System 2 规划，System 1 行动。好处：慢速 VLM 级规划不阻塞快速控制；System 1 保持小以降低延迟。

GR00T N1.7（2025 年末）改进了数据缩放。GR00T 使用来自 Omniverse 的 sim-to-real 数据微调。

### Open X-Embodiment

训练数据。RT-X（2023 年 10 月）组装了覆盖 22 种机器人 100 万条轨迹的 22 个数据集。Open X-Embodiment 是每个人使用的语料：

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table。
- 每个样本：（机器人状态、摄像头视图、指令、动作序列）。
- 训练卫生：统一动作空间、归一化关节范围、调整摄像头。

OpenVLA 和 π0 在 Open X-Embodiment 上训练。通过 100-1000 个任务特定演示的 LoRA 微调弥合与任何特定机器人的领域差距。

### 联合微调 vs 仅机器人训练

联合微调将网络 VQA 数据与机器人轨迹混合。比例很重要：太多 VQA 模型忘记动作；太多机器人数据模型失去通用知识。

RT-2 的比例：约 1:1。OpenVLA：网络与机器人约 0.5:1。π0：类似。精确比例是随数据集大小调优的超参数。

仅机器人训练产生在分布外指令上失败的任务特定模型。联合微调是"拿起红色方块（演示中）"和"拿起从左数第三大的物体（新颖措辞）"之间的区别。

### 安全和动作限制

每个生产 VLA 配备：

- 硬关节限制（不能超过规格扭矩）。
- 速度限制（软裁剪）。
- 工作空间边界（末端执行器不能离开桌面）。
- 新任务的人机协作审批。

这些位于 VLA 之外作为控制层检查。VLM 的输出是建议，不是命令。

## 使用它

`code/main.py`：

- 实现 256 分箱动作 token 化和反 token 化。
- 基于 DCT + 量化描绘 FAST tokenizer。
- 对比（离散分箱、FAST、连续流）每个动作步骤的 token 数。
- 打印 RT-2 → OpenVLA → π0 → GR00T 的谱系摘要。

## 练习

1. 30 Hz 控制速率的 10-DOF 手臂。256 分箱的离散分箱 token 化每秒发出多少个 token？7B VLM 能跟上吗？

2. FAST token 化将 30 步轨迹压缩到约 10 个 token。如果轨迹有高频运动（如打鼓），用户会损失什么？

3. π0 的流匹配头约 5 步去噪。对比 OpenVLA 4-5 Hz 自回归解码的吞吐量。

4. GR00T 的 System 1 / System 2 分离映射到卡尼曼。提出一个不同的分离（System 3？）可能有助于双足行走。

5. 阅读 Open X-Embodiment 第 4 节关于数据集策展。说出防止领域泄漏的三条策展规则。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| VLA | "视觉-语言-动作" | 接收图像 + 指令并输出动作命令的模型 |
| Action tokenization（动作 token 化） | "离散分箱" | 将连续关节目标每维量化为 256 个分箱，每个是一个词汇 ID |
| FAST tokenizer | "频域动作 token" | 通过 DCT + 量化将 30 步轨迹压缩到约 10 个 token |
| Co-fine-tune（联合微调） | "混合网络 + 机器人" | 在网络 VQA 数据旁训练机器人演示以保留通用知识 |
| Flow-matching action head（流匹配动作头） | "π0 连续输出" | 通过整流流输出 50 步动作序列的小型 Transformer |
| System 1 / System 2 | "双系统控制" | 大型 VLM 缓慢规划，小型动作头快速行动；GR00T 模式 |
| Open X-Embodiment | "RT-X 数据集" | 100 万条轨迹的跨机器人数据集；训练语料 |
