# MIO 与任意到任意流式多模态模型

> GPT-4o 交付了大多数开源模型无法复制的产品：一个能听语音、看视频并实时语音回复的代理。到 2024 年底，开源生态的答案是 MIO（Wang 等人，2024 年 9 月）。MIO 对文本、图像、语音和音乐进行 token 化，在交错序列上训练一个因果 Transformer，并生成任意模态到任意模态。AnyGPT（Zhan 等人，2024 年 2 月）是概念验证；MIO 是规模扩展版；Unified-IO 2（Allen AI，2023 年 12 月）是具有视觉 + 动作定位的表亲。本课解读任意到任意模式——四个 tokenizer，一个 Transformer，流式友好的解码。

**类型：** Learn
**语言：** Python（标准库，四模态 token 分配器 + 流式解码循环）
**前置课程：** Phase 12 · 11（Chameleon）、Phase 6（Speech and Audio）
**时间：** ~120 分钟

## 学习目标

- 设计一个容纳文本、图像、语音和音乐 token 而不冲突的共享词汇表。
- 对比 SEED-Tokenizer（图像）和 SpeechTokenizer 残差 VQ（语音）在压缩 + 重建权衡上的差异。
- 解释构建任意到任意生成的四阶段课程。
- 说出三种开源任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题

统一多模态模型声称容易但大规模构建困难。直到 2024 年，大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每一跳都损失信息、增加延迟并使训练复杂化。GPT-4o 的演示视频展示了亚秒级响应的单模型替代方案；开源系统落后数月。

工程挑战：

- 每种模态必须有 tokenizer，压缩足够无损以供重建，并以 Transformer 能消费的速率产生 token。
- 单一词汇表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。最少四万条目以上。
- 训练数据必须覆盖每种输入-输出对（文本→图像、图像→语音、语音→图像等），否则模型必须组合。
- 推理必须足够快地流式输出 token 以满足对话延迟（<500ms 首个音频字节时间）。

## 概念

### 四种模态的四个 tokenizer

MIO 的 tokenizer 堆栈：

- **文本**：标准 BPE，词汇表约 32000。
- **图像**：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 条目，每张图像 32×32 个 token。
- **语音**：SpeechTokenizer 残差 VQ（2023）——将 16kHz 波形编码为 8 个层次码本；第一层是粗粒度内容，后续层添加韵律和说话人身份。
- **音乐**：类似的残差 VQ（Meta 的 MusicGen / Encodec 系列），4-8 个码本。

每种模态产生整数 token。token 在共享词汇表中获得不相交的 ID 范围：

```
文本:   0..31999
图像:   32000..36095  (4096 个图像 token)
语音:   36096..40191  (4096 个语音基础 token，加上残差层)
音乐:   40192..48383  (8192 个音乐 token)
分隔符: 48384..48390  (<image>, <speech>, <music>, </...> 等)
```

总计：约 48k 词汇表。输入嵌入和输出投影覆盖全部。

### 流式解码

语音生成使用残差 VQ。Transformer 预测基础（第 0 层）语音 token；并行解码的残差量化器预测后续层。每个第 0 层 token 在 16kHz 下大约是 50ms 的音频。

流式模式：

1. 用户对着麦克风说话；实时音频 tokenizer 每 50ms 发出语音 token。
2. MIO 在 token 到达时消费（提示预填充 + 增量前向传播）。
3. 输出 token 在生成时流式输出；并行语音解码器以约 50-150ms 延迟将它们转换为音频样本。
4. 首个音频字节时间：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音 LLM 设计。Moshi 特别在单 GPU 上实现了 160ms 往返延迟。

### 四阶段课程

MIO 的训练课程：

1. **阶段 1——对齐**。大规模模态对语料：文本-图像、文本-语音、文本-音乐。每对使用自己的 token 词汇表段。训练共享词汇表。
2. **阶段 2——交错**。多模态交错文档（带图像 + 视频的博客、带转录的播客等）。训练跨模态上下文。
3. **阶段 3——语音增强**。额外音频数据以提升语音质量而不丢失文本能力。
4. **阶段 4——SFT**。跨模态指令微调：VQA、描述、旁白、语音到语音对话。

缺少某个阶段会退化特定能力：跳过阶段 2 模型失去跨模态上下文；跳过阶段 3 语音质量差。

### 视觉思维链

MIO 引入了视觉思维链：模型发出中间图像 token 作为推理步骤。对于"猫在爬树吗？"模型：

1. 发出渲染场景的 `<image>` token（来自输入图像或草图）。
2. 发出分析草图的文本。
3. 发出最终答案。

渲染的中间图像充当草稿纸。空间推理任务的基准提升。这个想法镜像了文本推理的思维链。

### 任意到任意的竞争者

- **AnyGPT**（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），类似设计。
- **Unified-IO 2**（arXiv:2312.17172）：添加视觉动作输出、深度、法线。更多任务多样性，更小规模。
- **NExT-GPT**（arXiv:2309.05519）：LLM + 模态特定扩散解码器。非单模型方法。
- **CoDi**（arXiv:2305.11846）：可组合扩散；通过共享潜在空间实现任意到任意。

MIO 最接近纯 token 任意到任意。AnyGPT 是其概念祖先。

### 延迟预算

对于对话产品，每个组件的延迟都很重要：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史）：8B 模型上约 100ms。
- 第一个输出 token：约 50ms。
- 并行残差 VQ + 语音解码器：约 100-150ms。

总首个音频字节时间：最低约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 按公开基准在 400-600ms 范围。

### 为什么任意到任意仍然困难

即使在 2026 年，开源任意到任意模型在两个轴上落后于闭源模型：

- **语音质量**。残差 VQ tokenizer 是有损的；对话语音听起来比 ElevenLabs 级声音更机械。
- **跨模态推理**。要求模型"唱出你看到的东西"仍然比纯视觉任务更容易失败。

这些是开放的研究问题。Qwen3-Omni（课程 12.20）是 2025 年最先进的开源尝试。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出花费时间的组件。

2. SpeechTokenizer 残差 VQ 使用 8 个码本。提出为什么并行解码残差层是必要的（vs 顺序的）以及它带来的延迟节省。

3. 你的词汇表有 32k 文本 + 4k 图像 + 4k 语音。添加 8k 音乐和约 10 个分隔符。隐藏维度 4096 下嵌入矩阵的参数成本是多少？

4. 视觉思维链发出中间图像。哪些问题受益？哪些被额外 token 损害？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技术并与 MIO 的视觉思维链对比。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Any-to-any（任意到任意） | "多模态输入/输出" | 单一模型接受并发出文本、图像、语音和音乐的任意方向 |
| Residual-VQ（残差 VQ） | "语音 tokenizer 堆栈" | 每层添加信息的多码本 token 化；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像编码" | MIO 使用的带 4096 条目码本的离散图像 tokenizer |
| Chain-of-visual-thought（视觉思维链） | "视觉草稿纸" | 模型在最终答案之前生成中间图像作为推理步骤 |
| Time-to-first-audio-byte（首个音频字节时间） | "TTFAB" | 从用户语音到首个音频输出的延迟；<500ms 为对话级体验 |
| Four-stage curriculum（四阶段课程） | "训练方案" | 对齐 → 交错 → 语音增强 → SFT，按此顺序 |
