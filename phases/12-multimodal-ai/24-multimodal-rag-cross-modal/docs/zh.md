# 多模态 RAG 与跨模态检索

> 视觉原生文档 RAG 是一个切面。生产级多模态 RAG 范围更广——跨文本、图像、音频和视频检索，用于行程规划（"找一家安静的、有自然光的素食早午餐"）、医疗分诊（"什么伤情匹配这张照片 + 这些笔记"）、电商（"类似这张自拍的穿搭，我的尺码"）和现场服务（"诊断这个引擎声音加零件照片"）等工作流。2025 年的三篇综述——Abootorabi 等人、Mei 等人、Zhao 等人——编纂了子问题：跨模态检索、检索融合、生成溯源、多模态评估。本课解读综述并设计生产流水线。

**类型：** Build
**语言：** Python（标准库，跨模态检索器 + 融合 + 溯源生成器）
**前置课程：** Phase 12 · 23（ColPali）、Phase 11（RAG 基础）
**时间：** ~180 分钟

## 学习目标

- 设计跨模态检索：文本→图像、图像→文本、音频→视频等。
- 对比三种融合策略：分数融合、基于注意力的融合、MoE 融合。
- 解释生成溯源：当来源是多种模态的混合时，"引用你的来源"是什么样的。
- 说出 2025 年三篇经典多模态 RAG 综述及其子问题分类法。

## 问题

单模态 RAG 是已解决的模式：嵌入查询，嵌入块，检索，填入 LLM。多模态 RAG 需要：

1. 多个检索头（每种模态需要兼容空间中的嵌入）。
2. 跨模态检索结果的融合。
3. 引用跨模态来源的生成溯源。
4. 覆盖跨模态信号的评估指标。

2025 年的综述都得出了相同的分类法。

## 概念

### 跨模态检索

给定模态 A 的查询，检索模态 B 的文档。三种模式：

1. **共享嵌入空间**。CLIP 和 CLAP 在共享空间中产生文本 + 图像 / 文本 + 音频嵌入。跨模态余弦相似度直接有效。限于 CLIP 训练过的对。

2. **每模态编码器 + 翻译**。文本编码器 + 图像编码器 + 一个小型翻译模块在空间之间映射。Gupta 等人的 Sen2Sen 和其他 2024 年设计。灵活但增加复杂度。

3. **VLM 作为编码器**。使用 VLM 的隐藏状态作为检索表示。VLM 支持的任何模态都有效。质量更高，成本更贵。

选择：文本+图像用 CLIP / SigLIP 2；文本+音频用 CLAP；前沿质量的跨模态用 VLM 隐藏状态。

### 融合策略

你检索到了 10 个结果：5 张图像、3 段文本、2 段音频。如何合并？

**分数融合**（最便宜）。每种模态有自己的检索器，各自返回分数。在模态内归一化分数然后求和。简单，通常有效。

**基于注意力的融合**。拼接所有检索到的项，让一个小型注意力网络加权它们。需要训练。

**MoE 融合**。门控网络路由到模态特定的专家。不同查询类型路由不同——视觉问题对图像权重更高。

生产默认：分数融合，略微偏向查询的主导模态。如果 A/B 测试显示在你的领域有明显优势，升级到 MoE。

### 生成溯源

LLM 应该引用哪个检索到的项支持了每个声明。对于多模态：

- 文本来源：标准引用 `[1]`。
- 图像来源：`[img 3]` 加简短描述。
- 音频：`[audio 2 at 0:34]`。

用溯源感知数据训练生成器：训练目标中的每个声明都标记了来源索引。推理时模型自然发出引用。

阅读三篇给你 2025 年春季的技术现状。大多数子问题仍是开放的。

### MuRAG——基础论文

MuRAG（Chen 等人，2022）是第一个多模态 RAG。从多模态知识库检索图像 + 文本，生成答案。在 VLM 浪潮之前展示了可行性。现代系统（REACT、VisRAG、M3DocRAG）建立在它之上。

### 生产级行程规划器示例

查询："找一家安静的、有自然光的素食早午餐。"

流水线：

1. **分解查询**。"安静" → 音频/评论关键词；"素食早午餐" → 菜单项；"自然光" → 图像特征。
2. **按模态检索**：
   - 文本检索评论："素食早午餐，安静氛围。"
   - 图像检索餐厅照片："自然光，通风。"
   - 音频检索环境声音片段："低分贝，无音乐。"
3. **融合分数**。每个餐厅有综合分数。
4. **Top-k 餐厅** → 带所有证据的 VLM 生成器 → 带引用的答案。

这远超文本 RAG。每种模态都添加了文本单独遗漏的信号。

### 代理式多模态 RAG

多跳：如果第一次检索没有返回高置信度答案，LLM 重新制定查询并再次检索。Phase 14 的代理式 RAG 模式在此适用。示例：

- 检索初始 top-10 → LLM 说"太嘈杂，过滤 <40 dB" → 重新检索。
- 检索图像 → LLM 看到一个有菜单 → 检索菜单文本 → 回答。

增加复杂度但处理了单次检索无法处理的查询。

### 评估

跨模态评估仍不成熟。常见代理指标：

- 每模态 Recall@k。
- 融合 top-k 准确率。
- 人工评判端到端满意度。
- 任务特定（完成的预订、完成的购买）。

没有跨所有模态的标准基准。大多数论文在领域特定任务上评估。

## 使用它

`code/main.py`：

- 三个模拟检索器（文本、图像、音频）在共享餐厅语料上运行。
- 分数融合以可配置权重组合模态分数。
- 生成器桩发出带引用的最终答案。
- 简单代理循环在置信度低时重新制定查询。

## 练习

1. 提出一个医疗分诊多模态 RAG：查询 = 伤情照片 + 文本症状。什么模态从什么知识库检索？

2. 分数融合是简单加权和。它有什么 MoE 融合避免的失败模式？

3. 阅读 Abootorabi 等人的分类法（第 3 节）。三个经典子问题是什么，它们如何映射到你选择的产品？

4. 为行程规划器多模态 RAG 设计评估规格。什么指标覆盖图像召回、音频召回和综合正确性？

5. 代理式多跳 RAG 每轮往返有延迟代价。在什么查询难度下准确率提升能证明延迟的合理性？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Cross-modal retrieval（跨模态检索） | "查询一种模态，检索另一种" | 文本查询检索图像；图像查询检索文本；需要共享空间或翻译器 |
| Score fusion（分数融合） | "合并分数" | 每模态检索分数的加权和；最简单的融合 |
| MoE fusion（MoE 融合） | "模态路由专家" | 门控网络为每个查询选择信任哪种模态的分数 |
| Grounded generation（溯源生成） | "引用你的来源" | 答案中的每个声明标记了来源索引 |
| MuRAG | "第一个多模态 RAG" | 2022 年确立多模态 RAG 模式的论文 |
| Agentic multi-hop（代理式多跳） | "重新制定并重试" | LLM 在首轮置信度低时重新查询检索器 |
