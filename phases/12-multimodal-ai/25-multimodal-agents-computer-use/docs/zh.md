# 多模态代理与计算机使用（Capstone）

> 2026 年的前沿产品是一个多模态代理：读取截图、点击按钮、导航 Web UI、填写表单并端到端完成工作流。SeeClick 和 CogAgent（2024）证明了 GUI 定位原语。Ferret-UI 扩展到移动端。ChartAgent 引入了图表的视觉工具使用。VisualWebArena 和 AgentVista（2026）是前沿追逐的基准——即使是 Gemini 3 Pro 和 Claude Opus 4.7 在 AgentVista 的困难任务上也只得分约 30%。本 capstone 汇集了 Phase 12 的每条线索：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、定位（坐标输出）、长视野记忆和评估。

**类型：** Capstone
**语言：** Python（标准库，动作 schema + 代理循环骨架）
**前置课程：** Phase 12 · 05（LLaVA）、Phase 12 · 09（Qwen-VL JSON）、Phase 14（Agent Engineering）
**时间：** ~240 分钟

## 学习目标

- 设计多模态代理循环：感知→推理→行动→观察→重复。
- 构建 VLM 可以 JSON 形式发出的 GUI 定位输出 schema（点击坐标、输入文本、滚动、拖拽）。
- 对比仅截图代理 vs 无障碍树代理 vs 混合代理。
- 在 VisualWebArena 小型切片上设置多模态代理基准评估。

## 问题

订票网站工作流："帮我找 4 月 15 日飞东京的航班，靠窗座位，800 美元以下，预订它。"

多模态代理需要：

1. 截取浏览器截图。
2. 将截图 + URL + 目标解析为计划。
3. 发出结构化动作：点击（在 x,y）、输入"Tokyo"（在元素 E）、向下滚动、选择（单选按钮）。
4. 将动作应用到浏览器。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是一次多模态 VLM 调用。VLM 输出必须是可解析的 JSON。错误在步骤间累积，因此恢复很重要。

## 概念

### GUI 定位——原语

GUI 定位是：给定截图和自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是第一个大规模开源结果：在合成 + 真实 GUI 数据上微调 VLM，以纯文本 token 形式输出坐标。有效。

CogAgent（arXiv:2312.08914）添加了 1120×1120 高分辨率编码以处理密集 UI。Web 导航上约 84%。

Ferret-UI（arXiv:2404.05719）聚焦移动端 UI，与 iOS 无障碍数据集成。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc` 有助于恢复：如果坐标在截图间漂移，语义提示让系统可以重新定位。

### 动作 schema

典型动作 schema 有 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

代理每步发出一个动作。浏览器包装器执行并返回新状态。

### 仅截图 vs 无障碍树

两种输入模式：

- **仅截图**：完整图像，无结构信息。最通用；适用于任何应用。
- **无障碍树**：结构化的 DOM / iOS 无障碍信息。定位更可靠；在树可用时有效。
- **混合**：两者都用，树作为原子动作的可靠定位器，截图提供语义上下文。

生产代理尽可能使用混合模式。浏览器自动化（Selenium + 无障碍）始终有树；桌面应用有时有。

### 长视野记忆

20 步工作流产生 20 张截图。VLM 的上下文很快填满。三种压缩策略：

- **摘要链**：每 5 步后，总结已发生的事情，丢弃旧截图。
- **跳帧**：保留第一张、最后一张和每第 3 张截图。
- **工具记录日志**：执行动作，保留已执行操作的文本日志；不再查看旧截图。

Claude 的 computer-use API 使用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）引入了用于图表理解的视觉工具使用：裁剪、缩放、OCR、调用外部检测。代理可以输出"裁剪到区域 (100, 200, 300, 400) 然后调用 OCR"作为工具调用。工具返回文本；VLM 继续推理。

此模式可泛化：集合标记提示、区域注释和外部检测工具都适用相同的"输出工具调用，接收结构化响应"schema。

### 2026 年基准

- **ScreenSpot-Pro**。约 1000 张 Web 截图上的 GUI 定位。开源 SOTA Qwen2.5-VL-72B 约 85%。前沿约 90%。
- **VisualWebArena**。端到端 Web 任务（商店、论坛、分类广告）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- **AgentVista**（arXiv:2602.23166）。2026 年最难的基准。跨 12 个领域的真实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- **WebArena / WebShop**。较老的基准；已被前沿饱和。

### 为什么仍然困难

代理性能瓶颈：

1. **细粒度视觉定位**。"点击小 X"在移动端分辨率下经常失败。
2. **长视野规划**。10 个动作后，代理偏离目标。
3. **错误恢复**。当点击失败（点错按钮），检测 + 恢复很少在训练数据中。
4. **跨页面上下文**。在标签页间跳转或长表单丢失状态。

研究方向：记忆架构、显式重规划、多模态验证（截图匹配以确认动作成功）。

### Capstone 构建

Capstone 任务：构建一个计算机使用代理：

1. 读取预订网站模拟页面的 HTML + 截图。
2. 规划多步骤序列：搜索→选择→填写表单→提交。
3. 发出匹配动作 schema 的 JSON 动作。
4. 在固定的 10 任务切片上评估。

课程提供了易于扩展到真实浏览器的脚手架代码。

## 使用它

`code/main.py` 是 capstone 脚手架：

- 动作 schema JSON 定义（10 种动作）。
- 模拟浏览器状态为字典。
- 代理循环骨架：接收状态、发出动作、应用、循环。
- 10 任务迷你基准（合成页面）以测量端到端成功率。
- 动作失败时的错误恢复钩子。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| GUI grounding（GUI 定位） | "点击坐标" | 模型在截图上输出指令目标的 (x,y) |
| Action schema（动作 schema） | "工具定义" | 有效动作的 JSON 描述（点击、输入、滚动、拖拽） |
| Accessibility tree（无障碍树） | "结构化 DOM" | 来自浏览器/iOS API 的机器可读 UI 层次结构 |
| Hybrid agent（混合代理） | "截图 + 树" | 同时使用图像和结构化信息；比单独使用更可靠 |
| Visual tool use（视觉工具使用） | "缩放/裁剪/检测" | 代理在计划中调用外部视觉工具（OCR、检测） |
| Summary-chain（摘要链） | "记忆压缩" | 定期文本摘要替代长截图历史 |
| VisualWebArena | "端到端 Web 基准" | 2024 年端到端 Web 任务基准 |
| AgentVista | "2026 困难基准" | 12 领域真实工作流；即使 Gemini 3 Pro 也仅约 30% |
