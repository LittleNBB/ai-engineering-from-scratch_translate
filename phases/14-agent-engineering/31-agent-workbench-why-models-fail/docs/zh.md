# 智能体工作台工程：为什么能力强的模型仍然失败

> 一个能力强的模型是不够的。可靠的智能体需要一个工作台（Workbench）：指令、状态、范围、反馈、验证、审查和交接。去掉这些，即使是前沿模型也会产出不可安全交付的工作。

**类型：** Learn + Build
**语言：** Python（标准库）
**前置课程：** Phase 14 · 01（Agent Loop），Phase 14 · 06（Failure Modes）
**时间：** ~45 分钟

## 学习目标

- 区分模型能力与执行可靠性。
- 列出决定智能体能否交付的七个工作台表面。
- 在一个小型仓库任务上对比仅提示运行与工作台引导运行。
- 生成一份失败模型报告，将每个缺失的表面映射到其导致的症状。
